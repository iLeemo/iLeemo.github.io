JS数组中的Sort( )和Fliter( )

发表于 2016-05-03

最近发现自己对数组的使用可以说是相当不了解，很奇怪，按道理说，数组应该是最常用的一个类型了，很用来用去，发现还是了解的太少了。
前两天看百度IFE，有这么一个问题：给你如下的数组，让你找出数值大于60的城市，并按由大到小的顺序输出。

一般思路就是遍历数组，取大于60的值，存入新数组，排序，输出，后来看到这一个解法。

这段写的挺有意思，它调用了三个方法filter()、sort()和forEach()。来看看这三个方法的功能吧：

filter( )

arr.filter(callback[, thisArg])方法的作用为：对数组中的每一项运行给定函数，返回该函数会返回 true 的项组成的数组。 它会进行三步工作，运行函数，判断是否为true；是的话存入到返回数组中。

参数

用来测试数组的每个元素的函数。调用时使用参数 (element, index, array)。返回true表示保留该元素（通过测试），false则不保留。

thisArg
可选。执行 callback 时的用于 this 的值

callback 被调用时传入三个参数：

  1. 元素的值
  2. 元素的索引
  3. 遍历的数组

拿上面的代码来说：

aqiData.filter(function(i){return i[1] > 60;})

其实就是一个数组，其中的参数i，代表着数组中的每一项，由于是数组嵌套，因此其后面加个索引1，即数值所在的位置，然后返回所有数值大于60的数组。

注意

需要注意的是，filter( )并不会改变原来数组的值，它会创建一个新的数组，且其所查找的值，在被调用之初已经确定了，即，如果在后面更改了原数组的值，如果不再次重新调用filter( )方法，那么原来得到的数组不会改变。
